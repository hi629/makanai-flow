# 筋トレAIアプリ（DBなしMVP）Codex向け仕様書

本仕様書は、**既存の献立アプリ（PlanEat / MAKANAI系）の雛形を流用**し、
**バックエンドDBを使わずに動作する「1週間筋トレメニュー生成アプリ」**へ変更するための
**Codex（Cursor / GPT-4系）向け実装指示書**である。

---

## 0. 基本方針（最重要）

- バックエンドDBは **使用しない**
- ログイン・ユーザーアカウントは **不要**
- データは **端末ローカルのみ保存**
- AIは **週1回 or 再生成時のみ呼び出す**
- 既存の「献立」「食材」「冷蔵庫」概念は **すべて筋トレ概念へ置換**する

---

## 1. 既存献立アプリからの概念置換マップ

| 旧（献立アプリ） | 新（筋トレアプリ） |
|---|---|
| 献立プラン | 週間トレーニングプラン |
| 食材入力 | 体情報・環境入力 |
| 冷蔵庫 | 自分の身体 / トレーニング環境 |
| 今日の献立 | 今日のトレーニング |
| 買い物リスト | 必要器具・注意点 |
| 作る気力 | 体調レベル |

---

## 2. 対象技術スタック

- React Native + Expo
- ローカルDB：SQLite or WatermelonDB
- AI：ChatGPT API（ステートレス）
- バックエンドサーバー：**なし**

---

## 3. 画面構成（MVP）

既存の献立アプリ画面構成を以下に置換する。

1. 初期アンケート画面（プロフィール入力）
2. 週間プラン画面（7日分一覧）
3. 今日のトレーニング画面
4. 履歴・達成状況画面（簡易）
